В редакторе должно быть несколько разделов

- Скелетная анимация, выбираем персонажа из списка и работает над анимацией.
Персонажа можно менять во время анимации. То есть анимация не должно быть привязана к персонажу,
она должна хранится отдельно и накладываться на персонажа.

Просмотр иерархии сцены в этом режиме посути не нужен, нам только персонаж доступен.

Таймлайн должен работать с ключами/параметрами объекта и не привязываться к ригу.
То есть ему без разницы что мы анимируем, блендшейп, кость или что угодно.

Анимации могут лежать в папках, то есть должна быть иерархическая система.

Убрать разделение на фута или нет, это будут отдельные персонажи.

- Репозиторий объектов по категориям. Сюда просто будут загружаться модельки которые
потому будут доступны в других редакторах. Пока на вход подается категория, имя и модель.
Потом прикрутить нужно теги для сортировки и поиска по моделям. При добавлении объекта
нужно генерировать ему уникальный id.

- Редактор клипов. Здесь каждый объект имеет свой таймлайн. Есть иерархия сцены.
Какой объект выбираем с тем объектом и работаем. Выбрали персонажа и у него открывается
таймлайн где фигарим анимацию. Таймлайн у всех будет глобальными. То есть все объекты
синхронно анимируются. Так как в целом этот функционал не будет заточен на что-то длинное.
На сцену из репозитория должны иметь добавить любой объект.

Размер таймлайна регулируется глобально, как и номер кадра. То есть при переходе на 100й кадр,
все в сцене перейдут на 100й кадр. В файл при этом сохранятся все объекты и всех их анимации.

Так же добавить проигрывание звуков.

- Репозиторий звуков по категориям, тоже самое что с объектами только для звуков.

Заметки по архитектуре

Надо создать свой объект и пользоваться им, а не три джесом.
Каждый объект может принимать ключ и применять его по своему усмотрению.
С каждого объекта может считывать ключ и получать его значение.
